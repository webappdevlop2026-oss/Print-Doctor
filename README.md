# প্রিন্ট ডাক্তার — সেটআপ নির্দেশনা (GitHub + Vercel + Google Gemini ফ্রি ভার্সন)

এই ওয়েবসাইট **Google Gemini API** ব্যবহার করে, যেটা সম্পূর্ণ ফ্রি — কোনো কার্ড লাগে না, কোনো মেয়াদ শেষ হয় না। এটা এখন **Vercel**-এর জন্য কনফিগার করা।

## ধাপ ১: ফ্রি API Key নিন
1. https://aistudio.google.com/apikey এ যান
2. আপনার Google (Gmail) অ্যাকাউন্ট দিয়ে লগইন করুন
3. "Create API key" বাটনে ক্লিক করুন
4. একটা নতুন Google Cloud প্রজেক্ট বানাতে বললে "Create" চেপে এগিয়ে যান
5. তৈরি হওয়া key টা কপি করে রাখুন (এটা `AIza...` দিয়ে শুরু হবে)

**কোনো কার্ড, কোনো বিলিং সেটআপ লাগবে না।**

## ধাপ ২: GitHub-এ পুশ করুন
1. GitHub-এ একটা নতুন রিপোজিটরি বানান (যেমন: `print-doctor`)
2. এই পুরো ফোল্ডারটা (index.html, api/chat.js, README.md সবকিছু) সেই রিপোতে push করুন:
   ```
   git init
   git add .
   git commit -m "Print Doctor website"
   git branch -M main
   git remote add origin https://github.com/আপনার-ইউজারনেম/print-doctor.git
   git push -u origin main
   ```

## ধাপ ৩: Vercel-এ ডিপ্লয় করুন
1. https://vercel.com এ যান, GitHub দিয়ে লগইন করুন
2. "Add New..." → "Project" ক্লিক করুন
3. আপনার `print-doctor` রিপোজিটরি সিলেক্ট করে "Import" করুন
4. Framework হিসেবে "Other" থাকবে — কিছু পরিবর্তনের দরকার নেই, ডিফল্ট সেটিংসই ঠিক আছে
5. "Environment Variables" অংশে একটা ভ্যারিয়েবল যোগ করুন:
   - Key: `GEMINI_API_KEY`
   - Value: আপনার কপি করা key টা বসান
6. "Deploy" বাটনে ক্লিক করুন

ব্যস, কয়েক সেকেন্ডের মধ্যে সাইট লাইভ হয়ে যাবে এবং একটা লিংক পাবেন (যেমন `print-doctor.vercel.app`)।

## ধাপ ৪: টেস্ট করুন
সাইট খুলে চ্যাট বক্সে একটা সমস্যা লিখে পাঠান। AI এখন ফ্রিতেই উত্তর দেবে।

## পরবর্তীতে কোড পরিবর্তন করলে
GitHub রিপোতে নতুন commit push করলেই Vercel স্বয়ংক্রিয়ভাবে নতুন ভার্সন ডিপ্লয় করে দেবে — আলাদা করে কিছু করা লাগবে না।

## ফ্রি লিমিট কতটুকু?
- প্রতিদিন কয়েকশো মেসেজ (একটা ছোট ব্যবসার জন্য যথেষ্ট বেশি)
- মিনিটে কয়েকটা মেসেজ পাঠানো যাবে
- হঠাৎ অনেক বেশি গ্রাহক একসাথে ব্যবহার করলে সাময়িক "একটু পর চেষ্টা করুন" মেসেজ আসতে পারে — সেটা স্বাভাবিক, একটু পরেই আবার কাজ করবে

## একটা কথা মনে রাখবেন
ফ্রি টিয়ারে গ্রাহকের লেখা মেসেজ Google তাদের মডেল উন্নত করতে ব্যবহার করতে পারে (এটা Google-এর শর্ত)। কোনো ব্যক্তিগত/স্পর্শকাতর তথ্য এখানে না দেওয়াই ভালো — যদিও প্রিন্টার সমস্যার আলোচনায় এটা সাধারণত কোনো সমস্যা না।

## PWA (মোবাইল অ্যাপের মতো ইনস্টল করা)
সাইটটা এখন একটা **Progressive Web App (PWA)** — মানে মোবাইল ফোনে ব্রাউজার থেকে "হোম স্ক্রিনে যোগ করুন" করলে এটা একদম নিজস্ব অ্যাপের মতো আইকনসহ খুলবে, ব্রাউজারের অ্যাড্রেস বার ছাড়াই।

**গ্রাহক যেভাবে ইনস্টল করবেন:**
- **Android (Chrome):** সাইট খুলে উপরে ডান দিকে "⋮" মেনু → **"Add to Home screen"** / **"Install app"**
- **iPhone (Safari):** সাইট খুলে নিচের Share বাটন (□↑) → **"Add to Home Screen"**

ইনস্টল করার পর আইকনে ক্লিক করলে সরাসরি অ্যাপ খুলবে, ব্রাউজার UI ছাড়াই। ইন্টারনেট দুর্বল থাকলেও মূল পেজটা (app shell) দ্রুত খুলে যাবে — শুধু AI-এর নতুন উত্তরের জন্য ইন্টারনেট লাগবে।

**নতুন যোগ হওয়া ফাইল:**
```
manifest.json         → অ্যাপের নাম, আইকন, রং ইত্যাদি বলে দেয়
sw.js                  → সার্ভিস ওয়ার্কার, অফলাইনেও অ্যাপ খোলার জন্য
icons/icon-192.png     → অ্যাপ আইকন (ছোট সাইজ)
icons/icon-512.png     → অ্যাপ আইকন (বড় সাইজ)
icons/icon-maskable-512.png → Android-এর অ্যাডাপ্টিভ আইকনের জন্য
vercel.json            → sw.js যেন সবসময় সবশেষ ভার্সন লোড হয় তা নিশ্চিত করে
```

সবগুলো ফাইল GitHub রিপোতে ঠিক এই একই ফোল্ডার গঠন মেনে আপলোড করতে হবে (বিশেষ করে `icons` ফোল্ডারের ভেতরে icon ফাইলগুলো রাখা জরুরি)।

## ফোল্ডার গঠন
```
index.html                    → মূল ওয়েবসাইট
api/chat.js                    → API key নিরাপদে রেখে Gemini-এ কল করে (Vercel serverless function)
manifest.json                  → PWA কনফিগারেশন
sw.js                           → PWA সার্ভিস ওয়ার্কার
icons/icon-192.png              → অ্যাপ আইকন
icons/icon-512.png              → অ্যাপ আইকন
icons/icon-maskable-512.png     → অ্যাপ আইকন (adaptive)
vercel.json                     → ক্যাশিং কনফিগারেশন
README.md                       → এই নির্দেশনা
```

## সমস্যা হলে
- "সংযোগে সমস্যা হয়েছে" দেখালে → Vercel-এর Project → Settings → Environment Variables-এ `GEMINI_API_KEY` ঠিকমতো বসানো আছে কিনা চেক করুন, এরপর নতুন করে redeploy করুন (Deployments ট্যাব → সর্বশেষ ডিপ্লয়মেন্টের পাশে "..." → "Redeploy")
- Vercel-এর Project → "Logs" বা "Functions" ট্যাব থেকে `api/chat` ফাংশনের লগ দেখলে আসল error message পাবেন
- environment variable যোগ/পরিবর্তন করার পর সবসময় redeploy করতে হবে, নাহলে পুরনো ভ্যালু ব্যবহার হবে
